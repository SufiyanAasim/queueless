# Support

## Where to get help

| Need | Go to |
|------|-------|
| Set up and run locally | [RUNNING.md](RUNNING.md) |
| Something is broken | [docs/Troubleshooting.md](docs/Troubleshooting.md), then open a [bug report](https://github.com/SufiyanAasim/queueless/issues/new/choose) |
| How does X work? | [docs/](docs/) — architecture, API, deployment, guides |
| Feature idea | Open a [feature request](https://github.com/SufiyanAasim/queueless/issues/new/choose) |
| Security issue | **Privately** — see [SECURITY.md](SECURITY.md). Never a public issue. |
| Anything else | Email <sufiyanaasim@outlook.com> |

## Before opening an issue

1. Search existing issues — your problem may already be tracked.
2. Reproduce on the latest release (`v1.7.0 "Cosmos"` or newer).
3. Collect the browser console output and/or backend logs.
4. Note your environment: browser, OS, and whether you run locally or on
   Vercel + Render.

## Response expectations

This is an actively maintained solo project. Issues are typically triaged
within a few days; security reports within 72 hours.
