# QueueLess — v1.4.5

**Codename: Zenith — Intelligent Collaboration**

## 📖 Overview

Zenith transforms QueueLess from a queue manager into an **intelligent
operational workspace**: a grounded AI assistant, internal team messaging,
a notification center, secure sharing, shared files, and role-based access
control — all running on the free Firebase Spark plan.

Building on Polaris's flowing queues, this release adds the collaboration
layer: the people running the queue can now talk, share, and ask questions
of their own data without leaving the app.

## 🎯 Objectives

- Ship an AI assistant that **never fabricates operational data**.
- Build internal messaging with no external services.
- Centralise notifications behind an application event bus.
- Introduce role hierarchy and an audit trail.

## ✨ Added

### 🤖 AI Assistant (RAG)

- Floating launcher + full workspace (`/assistant`) with saved
  conversations (pin / rename / delete / export).
- **Provider abstraction**: zero-config grounded default;
  OpenAI / Groq / OpenRouter / Ollama / Gemini opt-in via `AI_PROVIDER`,
  with automatic failover to grounded.
- Retrieval-Augmented Generation over verified backend data only —
  rate-limited 20/min/user.

### 💬 Internal messaging

1:1 + group chat, team directory with presence, emoji reactions, read
receipts ("Seen"), inline attachments ≤ 256 kB.

### 🔔 Event bus + Notification Center

In-process event bus (`token.referred`, `queue.created`,
`message.sent`, …) feeding a header bell with unread badge and a dedicated
`/notifications` screen with filters.

### 🔗 Secure sharing & 📁 shared files

Expiring capability links + QR + printable view; drag-and-drop file
sharing ≤ 2 MB stored in RTDB (Spark-plan friendly).

### 🛡️ RBAC + audit log

Super Admin > Admin > Manager > Staff with `requireRole` middleware, role
management UI, and an append-only audit trail (`/admin/audit`).

## 🏛️ Architecture Progress

Zenith establishes the **signal-node pattern**: content is never
client-readable in RTDB; clients subscribe to content-free signal nodes
and refetch through the JWT API, which enforces membership/RBAC
server-side. Every collaboration feature since is built on it.

Known limitations at this release:

- Attachments/files bounded (256 kB / 2 MB) by the Spark plan.
- Assistant answers are grounded-only unless an LLM key is configured.
