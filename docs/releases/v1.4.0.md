# QueueLess — v1.4.0

**Codename: Polaris — Relay**

## 📖 Overview

Polaris teaches the queue to *flow*: live tokens transfer between counters
without losing their place, administrators shape their own queues instead
of fixed industry lists, and predictions come from trained models.

Building on Nebula's proactive automation, this release rethinks the queue
as a network of counters rather than a single line.

## 🎯 Objectives

- Transfer a live token between counters without re-queuing.
- Let admins create and manage their own queues.
- Replace fixed averages with trained predictions.

## ✨ Added

### 🔀 Token referral

Transfer a live token between counters (e.g. OPD → Eye Specialist); it
keeps its number, records a referral trail, is served as priority at the
destination, and never auto-expires mid-transfer.

### 🧩 Custom queue management

Full CRUD with capacity, token prefix, working hours, staff assignment,
per-queue analytics, and delete safeguards — across dedicated list /
create / manage screens.

### ⏱️ Live status

"Serving for" timer, **Next in Queue** indicator, and optional patient
name on tokens.

### 🔮 Predictive insights

Trained artefact (GradientBoosting service-time + IsolationForest
anomalies + seasonal arrivals) with an explainable heuristic fallback —
deterministic and cold-start safe. Full token lifecycle mirrored to
MongoDB.

### 🎖️ Credits page

Interactive credits screen introduced at `/credits`.

## 🔄 Changed

- "Industry profile" renamed **Industry Type**, with richer default queues
  for every industry.
- Frontend API client modularised (`services/api/*`) with a barrel
  re-export — zero breaking imports.

## 🏛️ Architecture Progress

Polaris decouples queues from industry profiles: custom queues take
precedence wherever defined, industry defaults remain the fallback. The
prediction service establishes the *artefact-with-fallback* ML pattern.

Known limitations at this release:

- Referral requires staff/admin action (no customer self-transfer).
