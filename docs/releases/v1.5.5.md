# QueueLess — v1.5.5

**Codename: Pulsar — Insight**

## 📖 Overview

Pulsar is an input-polish milestone: clock-style time pickers wherever a
time is selected, and a proper city selector at organisation setup.

Building on Orion's data-quality fixes, this release makes the *inputs*
match the quality of the data behind them.

## 🎯 Objectives

- Use native clock-style time pickers for every time field.
- Replace the free-text location with a structured city selection.

## ✨ Added

### 🏙️ City selection at setup

"Configure your queue" now offers a **city dropdown** (major cities plus a
custom "Other" input) alongside the organisation name — shown in the
status bar and on printed tokens.

### 🕰️ Clock-style time pickers

Queue **working hours** (create/edit queue) use the same native clock
picker as the daily auto-reset field — no more free-text `HH:MM`.

## 🔄 Changed

- Saving settings broadcasts the new configuration app-wide — the status
  bar (org · city) updates instantly without a reload.

## 🏛️ Architecture Progress

Pulsar standardises the input layer: structured pickers over free text,
one pattern per data type.

Known limitations at this release:

- `location` remains a plain string in config (no geo-validation).
