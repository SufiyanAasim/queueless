# QueueLess — v1.2.0

**Codename: Nova — Sight**

## 📖 Overview

Nova is the first stable release of QueueLess — a cloud-native,
token-based queue management system that replaces paper tokens with a
real-time browser experience. No app install, no accounts for customers.

This is where the product gained *sight*: live queue state visible to
every customer, admin, staff member, and display board simultaneously.

## 🎯 Objectives

- Issue and track tokens entirely in the browser, in real time.
- Give admins full queue control from a secure portal.
- Ship on free-tier cloud infrastructure end-to-end.

## ✨ Added

### 🎫 Core queue system

Walk-in token issuance with email tracking links · priority tokens ·
expiry and no-show handling · customer tracking page (`/token/:id`) with
live position, ETA, QR, WhatsApp share, push notifications, and confetti
on call.

### 🛠️ Admin portal

Secure JWT login · pause/resume/reset · call next / skip ·
multi-service counters · **ML Auto Mode** (predicted call intervals) ·
analytics heatmap · staff management.

### 🧑‍💼 Staff portal + 📺 display board

Staff login (password or kiosk PIN) with per-counter view and presence;
full-screen TV board (`/display`) with now-serving per service and a live
clock.

## 🏛️ Architecture Progress

Nova establishes the architecture everything since builds on: React +
Vite on Vercel, Express on Render, Firebase RTDB as the read-only
real-time store (writes only through the API), MongoDB Atlas analytics,
and GitHub Actions CI/CD.

Known limitations at this release:

- Fixed industry service lists; no custom queues.
- No staff attribution on analytics events.
