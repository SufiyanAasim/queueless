# QueueLess — v1.6.5

**Codename: Aurora — Summit**

## 📖 Overview

Aurora is a refinement milestone — a credits-page overhaul, live
configuration broadcasting, layout fixes around the docked messaging tray,
a deployment-gap regression test, and Render deploy hardening.

Building on the docked-tray messaging surface introduced in Quasar, this
release focuses on finishing details: the settings you save appear
instantly, the status bar is never covered, and the credits page presents
the developer professionally.

## 🎯 Objectives

- Overhaul the Credits page presentation.
- Broadcast configuration changes app-wide without a reload.
- Fix layout collisions between fixed docks and the footer status bar.
- Guard the deployment gap with a regression test.
- Make Render deploys reproducible.

## ✨ Added

### 🧪 Deployment-gap regression test

New test: `GET /admin/queues/overview` must return `401` without auth and
`200` with a valid token — **never `404`** — guarding against a stale
backend deploy going unnoticed.

## 🔄 Changed

### 🎖️ Credits page overhaul

- Role subtitle simplified to **"Software Engineer"**; specialisms moved
  into chip tags.
- New **Frameworks & Tools** chip grid (React, Vite, Tailwind CSS,
  Node.js, Express, Firebase, MongoDB, scikit-learn, pandas,
  GitHub Actions).
- Removed the automatic `mailto:` redirect on the thank-you button.

### ⚡ Live config broadcasting

Saving settings dispatches a `queueless:config` event; every consumer
(footer status bar, queue labels) updates immediately — no reload.

### 🚀 Render deployment reliability

`render.yaml` build changed from `npm install` to `npm ci` for exact
locked dependency versions on every deploy.

## 🐛 Fixed

- Footer clearance (`pb-20`) so page content is never hidden behind the
  fixed messaging dock.
- The collapsed messaging toggle shrinks to its content width, removing a
  wide invisible tap target over the bottom-right corner.

## 🏛️ Architecture Progress

Aurora establishes the *live-config broadcast* pattern (custom DOM event →
`useAppConfig` subscribers) used by every settings-driven surface since.

Known limitations at this release:

- No automated frontend test suite (backend: 48 integration tests).
- Render free tier cold starts (~30 s after idle).
