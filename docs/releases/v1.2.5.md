# QueueLess — v1.2.5

**Codename: Comet — Alive**

*(Originally tagged v1.2.2; renumbered to v1.2.5 in the release
re-coding.)*

## 📖 Overview

Comet is a focused patch milestone that hardens the analytics pipeline,
fixes UI regressions, and improves admin tooling — proving the Nova
foundation *alive* under real traffic.

## 🎯 Objectives

- Make analytics redundant and self-refreshing.
- Ship a detailed, printable admin report.
- Clean up layout regressions from the initial release.

## ✨ Added

- **Detailed report page** (`/admin/report`) with an automated action plan
  and staffing recommendation.
- **Dual-write analytics** (MongoDB + CSV) with graceful fallback.
- **Auto-refresh analytics** every 30 s + manual refresh button.
- Global admin navigation bar (Dashboard, Analytics, Report, Settings).

## 🐛 Fixed

- Heatmap hours computed from real data (was a hardcoded range);
  per-service intensity scaling; dynamic chart labels.
- Hero layout at 100 % zoom; footer logo; live status card alignment.
- Vite peer-dependency conflict for clean Vercel builds; high-severity
  npm vulnerability patched.

## 🏛️ Architecture Progress

Comet introduces the dual-write analytics contract (MongoDB + CSV) that
the entire data-mining pipeline is built on.

Known limitations at this release:

- Analytics charts still show all services regardless of Industry Type
  (fixed in Orion).
