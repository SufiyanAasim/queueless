# QueueLess — v1.7.0

**Codename: Cosmos — LAN Connectivity & UI Polish**

## 📖 Overview

Cosmos is the current release of QueueLess — a connectivity-and-polish
milestone that brings first-class LAN support for kiosks and display
boards, an interactive-UI pass across the admin surface, automatic session
expiry, dark-mode readability fixes, and a professional repository
structure aligned with industry (and Metro Navigation System) conventions.

Building on the intelligent-workspace foundation established in Zenith and
refined through Orion, Pulsar, Quasar, and Aurora, this release focuses on
how the product *feels*: every screen is reachable, every panel behaves,
every number is live, and the repository reads like a professional
engineering project.

## 🎯 Objectives

This milestone was developed with the following goals:

- Make the app usable across a local network (kiosks, TV display boards).
- Remove every remaining static value from user-facing analytics.
- Make the UI interactive wherever an element implies an action.
- Introduce session expiry with a clear, friendly sign-out experience.
- Restructure the repository and documentation to professional standards.

## ✨ Added

### 📡 LAN connectivity

- New `npm run dev:lan` script binds the dev server to all network
  interfaces — any device on the LAN can open the app via the host's IP.
- Step-by-step LAN guide (API base URL + CORS setup) in `RUNNING.md` and
  `docs/Development.md`.

### 🕒 Session expiry

- New `useSessionExpiry` watchdog decodes the stored admin/staff JWT and
  checks every 30 seconds — expired sessions sign out immediately and land
  on the login screen with a **"Your session has expired"** notice.

### 🖱️ Interactive UI pass

- Dashboard metric tiles deep-link into Analytics with hover affordances.
- The SLA overdue alert bar is clickable (straight to Analytics).
- Hover/cursor feedback added wherever an element performs an action.

### 🏛️ Professional repository structure

- `.github/`: bug/feature/docs/question issue templates, security contact
  link, PR template, CODEOWNERS, grouped Dependabot configuration.
- `docs/`: Architecture, API reference, Deployment, Development,
  Admin/Staff/User guides, Troubleshooting, and per-release notes —
  flat **TitleCase** naming matching the Metro Navigation System repo.
- Root governance: CONTRIBUTING, SECURITY, CODE_OF_CONDUCT, SUPPORT,
  ROADMAP, RELEASE, `.editorconfig`, `.gitattributes`.

## 🔄 Changed

### 🚪 Auto-closing workspace panels

- The ✦ Assistant panel and 💬 messaging tray close automatically on
  screen switch (route change); the tray's closed state persists.

### 🧭 Hidden screens surfaced

- **AI Workspace**, **Notifications**, and **Shared files** links added to
  the ADMIN ▾ / staff account menus (desktop + mobile).

### 📈 No static analytics data

- Public `/config` now returns a live `avgServiceSeconds` (observed waits
  → configured default); the Take-a-Token wait preview uses per-queue
  averages → live org average. The last hardcoded `180 s` constant is gone.

### 👤 Author identity

- GitHub handle updated to **SufiyanAasim** across the app and docs; the
  Credits picture loads live from `github.com/SufiyanAasim.png`.

## 🐛 Fixed

### 🌒 Dark-mode readability

- Global overrides for `text-paper` variants inside flipped `bg-ink`
  surfaces — fixes invisible text in Smart Staffing, the docked messaging
  bar, and chat bubbles; light chips receive a dark tint.

## 🏛️ Architecture Progress

Session expiry establishes the client-side auth-lifecycle pattern
(decode → watch → graceful sign-out) on top of the existing JWT model,
and the docs restructure gives every subsystem a single authoritative
reference. Cosmos completes the arc started in Orion: trustworthy data
(Orion), polished inputs (Pulsar), a modern messaging surface (Quasar),
a governed repository (Aurora), and now a connected, interactive,
professionally documented product (Cosmos).

Known limitations at this release:

- No automated frontend test suite (backend: 48 integration tests).
- Free-tier hosting: Render cold starts ~30 s after idle.
- Shared files bounded at 2 MB (Firebase Spark plan — no Cloud Storage).
