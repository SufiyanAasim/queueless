# QueueLess — v1.3.0

**Codename: Eclipse — Crew**

## 📖 Overview

Eclipse is the account-management and priority-engine milestone — full
profiles and password flows for admins and staff, a smarter cross-counter
priority engine, eight new platform features, and a zero-vulnerability
dependency baseline.

Building on Comet's hardened analytics, this release grows the *crew*:
the people operating QueueLess get real accounts, and priority customers
get guaranteed treatment.

## 🎯 Objectives

- Give admins and staff proper account management.
- Enforce priority handling across every counter.
- Reach zero known dependency vulnerabilities.

## ✨ Added

### 👤 Account management

Admin & staff profiles, change-password flows, and account dropdown menus
(theme toggle, profile, password, sign out).

### 🚦 Priority queue engine

- Dedicated priority section above all counters.
- Regular queues auto-block while priority tokens wait — enforced on both
  frontend and backend (`409 PRIORITY_BLOCKING`).
- Cross-service `call-next-priority`.
- Priority tokens bypass the queue-paused gate.

### 🌟 Eight platform features

Display-board upgrades · live announcements · staff-dashboard upgrade ·
wait preview on Take-a-Token · token lookup · analytics charts ·
token notes · appointment booking.

## 🔒 Security

- `firebase-admin` 12 → 14 (clears the vulnerable
  gaxios/google-gax/teeny-request chain).
- Overrides for `uuid` (GHSA-w5hq-g745-h8pq) and `js-yaml` (merge-key DoS).
- **`npm audit`: 0 vulnerabilities** (was 8 moderate).

## 🐛 Fixed

Dark-mode contrast on `bg-ink` elements · Settings auth guard ·
401 interceptor scoped so cold starts don't wipe sessions ·
Rules-of-Hooks violations · authenticated CSV export ·
`loading` reference error on Home.

## 🏛️ Architecture Progress

Eclipse establishes role-segregated portals (admin vs staff JWT claims)
and the priority-blocking contract later reused by referral priority in
Polaris.

Known limitations at this release:

- Single admin-defined service list per industry; no custom queues yet.
