# QueueLess — v1.5.0

**Codename: Orion — Beacon**

## 📖 Overview

Orion is a data-quality milestone: analytics that respect your Industry
Type, a CSV export you can actually hand to an analyst, a proper No-show
button, and the closure of the deployment gap behind the live 404s.

Building on Zenith's intelligent workspace, this release makes the numbers
*trustworthy* — nothing stale, nothing unlabeled, nothing static.

## 🎯 Objectives

- Filter analytics to the current Industry Type / custom queues.
- Enrich the exported CSV with counter and organisation context.
- Give No-show / Skip a real button.
- Diagnose and close the live `Route not found` 404s.

## ✨ Added

### 🧾 Enriched analytics CSV

`queue_events.csv` now carries the **serving counter** (`staff_username`)
and **organisation name** alongside token number, service, and timestamps
— flowing through the Python pipeline as `served_by` / `org_name` columns.
Legacy-schema files are archived automatically (`*.legacy-<ts>`) so rows
never go ragged.

### 🧮 Chart clarity

Clarifying subtitle + empty-state on the Service Distribution chart.

## 🔄 Changed

### 🎯 Industry-aware analytics

Service Distribution and peak-hour charts show **only the current Industry
Type's services** (or your custom queues) — stale services from earlier
configurations (Forex/Loan after switching away from Bank) no longer
pollute the charts.

### ⊘ Proper No-show / Skip control

The skip action on admin and staff dashboards is a clearly styled button
(was an easy-to-miss text link).

## 🐛 Fixed

### 🧭 Deployment gap

Confirmed every "missing" route (`/admin/queues/overview`, `/admin/audit`,
`/admin/queue/refer/:id`, `/admin/predictions`, `/uploads`,
`/conversations`, `/notifications`) resolves in code — the live 404s were
a **stale backend on Render**, fixed by redeploying.

## 🏛️ Architecture Progress

Orion introduces the *allowed-services* filter shared by every analytics
surface, and the CSV schema-migration guard on the backend.

Known limitations at this release:

- Analytics page refreshes on a 30 s poll (dashboard is push-based).
